# 向晴简历｜闲鱼订单闭环上线配置

## 已实现的订单链路

1. 登录用户在网站选择套餐；
2. 服务端生成唯一网站订单号，状态为 `PENDING`，并绑定 Firebase UID；
3. 用户跳转到对应闲鱼商品付款，并在闲鱼聊天中发送网站订单号；
4. 卖家在 `/admin.html` 输入网站订单号、闲鱼订单号和实付金额；
5. 服务端校验套餐金额、订单状态和闲鱼订单号唯一性；
6. Firestore 在同一个事务中标记订单到账、发放额度/会员、标记订单 `FULFILLED`；
7. 用户付款页轮询订单状态，确认后显示“已到账 / 已发放”。

普通个人闲鱼卖家没有供当前外部网站直接订阅的卖家收款回调，因此“确认闲鱼到账”由卖家人工完成；订单校验、幂等、防重复发放和权益入账由系统自动完成。

## 你需要完成的 Firebase 配置

1. Firebase Console → Firestore Database → 以“生产模式”创建数据库（浏览器端无需直接读取 Firestore）；
2. Firebase Console → 项目设置 → 服务账号 → 生成新的私钥 JSON；
3. 私钥 JSON 只保存于本机，不要提交 Git，也不要发到聊天；
4. 将 JSON 转为一行 base64：

```bash
base64 < firebase-service-account.json | tr -d '\n'
```

5. 把结果保存到 Vercel 加密环境变量 `FIREBASE_SERVICE_ACCOUNT_BASE64`。

## Vercel 必需环境变量

| 变量 | 内容 |
|---|---|
| `FIREBASE_SERVICE_ACCOUNT_BASE64` | Firebase 服务账号 JSON 的 base64 内容 |
| `ADMIN_SECRET` | 卖家核款后台的强随机密钥 |
| `XIANYU_SINGLE_URL` | ¥2.98 单次下载商品链接 |
| `XIANYU_BASIC_URL` | ¥19.8 会员商品链接 |
| `XIANYU_PRO_URL` | ¥29.8 无限会员商品链接 |

当前代码已经内置以下公开商品链接作为默认值，Vercel 中不填写也能使用；以后商品重发时再用环境变量覆盖：

- ¥2.98 单次：`https://m.tb.cn/h.RvJF1Sb?tk=07vFgPDqfsx`
- ¥19.8 会员：`https://m.tb.cn/h.RvJw8Mt?tk=w9GtgPDrd8x`
- ¥29.8 无限会员：`https://m.tb.cn/h.RvJDEOQ?tk=bALjgPDs6dw`

生成后台密钥：

```bash
openssl rand -hex 32
```

环境变量添加后，必须重新部署 Vercel。

## 卖家使用方式

打开 `https://你的域名/admin.html`，只在确认闲鱼订单已经付款后填写：

- 后台密钥；
- 买家发来的网站订单号；
- 闲鱼卖家端的订单号；
- 闲鱼实付金额。

金额不一致、闲鱼订单号重复、网站订单不存在时，系统会拒绝发放。
